这几个问题拖了好些天了，正好没事干，就顺便完成吧。

泛型

先说怎么使用，在集合中Map<Integer>这就表示使用了泛型类。

那怎么定义的，点开map源码就会发现实在类名后面加上了通配符

Map<T>这个不一定准确，只是说大概意思。

搞明白了用法和定义。

那么就剩下使用范围，为什么要这样做，

那么这点就简单，先把泛型用法去掉，看看会发生什么。

以动态数组ArrayList为例子。

这是一个动态数组，所谓动态数组就是数组+自动扩容，数组又是同一种类型的集合，例如string数组，int数组。Java是静态语言，所以这一个数组内都是相同类型的。

上述可以理解吧，那么现在的问题是什么，如果没有泛型的话，那么ArrayList存放的是那种数据类型。

为什么有这个问题，那是因为要存入数据的话，内存是要开辟内存空间的，不同的类型所占据的内存空间不同，而且就算是数组的话，也是int[]来表明这里面存放的都是整形。

那是不是没有办法去搞了呢，不是，那样还可以将数据类型变成Object，毕竟所有的类都是继承自Object。

看似问题解决了，那么怎么确保存入的数据类型是一致的呢，就会发现，不行，搞不定，因为存放的数据都会变成object类型，不仅类型信息丢失了，而且还没有办法进行统一化操作。

那既然没有办法说数据类型不能为object，那么就剩下单独为每一种类型写一个方法。

且不说除了基本类型，还有自定义类型，写不过来，那么用的时候再写，就跟C语言没什么区别了。

而且来看自定义和原本的有什么区别。

public class StringArrayList {
    // 因为这种ArrayList只存String，所以不需要用Object[]兼容所有类型，只要String[]即可
    private String[] array;
    private int size;
    public void add(String e) {...}
    public void remove(int index) {...}
    public String get(int index) {...}
}
public class ArrayList<T> {
    private T[] array;
    private int size;
    public void add(T e) {...}
    public void remove(int index) {...}
    public T get(int index) {...}
}

其实写法是一样的，但是就可以针对所有的类型了，而不至于每种方法都写一遍。

看完上面可能觉得云里雾里，那不如这样去说，泛型其实就是模板，解决需要为每一种类型都写一个特定的方法。并且还不会丢失类型信息。

泛型方法同理。

补充

虽然写法和用法都很简单，原理也就是固定一个参数，用的时候传参就完了。但是深入了解Java底层原理的话，其实没这么简单，没有固定类型，那么写的类加载到内存的时候是怎么固定的，毕竟这是静态语言，底层其实就是用object数组接收，然后取出的时候强制转换，放入的时候进行校验。不过这些不用想自己怎么写，因为这都是编译之后才能看见的，自己该怎么写就怎么写。

枚举

额，这用法就是用enum代替class标识一个类。然后内容就是写字段即可。

看几个枚举类就懂了写法。

至于说原理，或者说咦，为什么多添加了几个信息，怎么下方还要定义字段，构造器里面还要写上这些字段的信息。并且还要多出@Get的方法。

/**
 * @author qiyu
 */
@Getter
public enum WeekDayEnum {
    MONDAY(1,"星期一"),
    TUESDAY(2,"星期二"),
    WEDNESDAY(3,"星期三"),
    THURSDAY(4,"星期四"),
    FRIDAY(5,"星期五"),
    SATURDAY(6,"星期六"),
    SUNDAY(7,"星期日");

    WeekDayEnum(Integer code, String desc) {
        this.code = code;
        this.desc = desc;
    }

    private final Integer code;
    private final String desc;
}
//用法
userDO.setRestDay(WeekDayEnum.FRIDAY);

倒不如这样去想，是因为例如Monday是weekdayenum的子类。所以才会有这样的写法。

那么问题来了，为什么这样做，写个常量不好么，其实主要的问题在于，校验，因为是子类，所以在定义方法的时候，可以用父类进行限制入参，并且还可以携带更多的信息，用常量的话，最多只能携带一个字符串的信息。但是当作一个类的话，就可以携带更多的字段。这里面还没表现出order的字段。

所以用法还有写法都挺简单，而这样写可以拥有更多的字段，这样就足够了。再多的信息就很杂乱了。

lambda表达式

String[] array = ...
Arrays.sort(array, new Comparator<String>() {
    public int compare(String s1, String s2) {
        return s1.compareTo(s2);
    }
});


import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        String[] array = new String[] { "Apple", "Orange", "Banana", "Lemon" };
        Arrays.sort(array, (s1, s2) -> {
            return s1.compareTo(s2);
        });
        System.out.println(String.join(", ", array));
    }
}

这里是arrays.sort的方法定义

也就是说这是要一个comparator接口的实现类。

而创建这个实现类，有三种，一种是在外部定义好接口实现类，然后直接传。第二种是匿名类，第三种是lambda表达式。

那么就可以仔细看这玩意儿 都有什么区别。然后再想想看缺少的东西可以在哪里得到补充。

然后在这里发现了什么，lambda等于匿名类对象。这个可以接受吧。

当然只是现阶段。

那么就看限制条件，限制能使用lambda的时候，这个接口只能是单方法接口。什么意思呢，就是这个接口只能定义一个抽象方法，静态方法和默认方法不算。

那为什么会这样。

主要原因在于，创建匿名类的时候，我们会有一个@override。并且把所有的接口方法都要实现。这一点才是关键。我们用匿名类的时候无所谓，多实现一两个函数的事。

但是lambda表达式呢，没有类名，没有方法名，入参类型也没有，如果这个接口内有两个抽象方法，那么请问这个时候的lambda表达式是实现的哪一个。这就是为什么lambda表达式只能实现单方法接口的原因。

上述是用lambda表达式等于匿名类对象。其实还是有所不妥，因为this的指向有问题。如果用lambda表达式，this指向的是调用lambda表达式的函数的this。但如果是匿名类对象，this指向的是匿名类的this。所以可以把lambda当作匿名类，但不是匿名类对象，

全局异常返回类

这个咋说呢，就是框架里面的内容，所谓的全局异常，其实就是用try catch捕捉异常的时候，可以捕捉自己定义的全局异常类，因为自定义异常是跑出来的子类。

为什么有这玩意儿，根本意义上是因为要处理异常信息，不然爆出来一大堆看不懂的异常，这谁能看得懂。
